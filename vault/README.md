# signalForge Vault

This vault is the human-readable operating layer for signalForge.

Use it to review system status, edit AI prompts, inspect generated outputs, and keep a daily command trail. Services mount this directory at `/vault`, so automation can read prompts and write notes without needing to know anything about Obsidian.

## Operating Folders

- `prompts/`: editable prompt templates used by services.
- `leads/`: lead notes and lead templates.
- `companies/`: company notes and company templates.
- `campaigns/`: campaign plans and outreach programs.
- `content/`: generated posts and drafts.
- `logs/`: daily operating logs.

## Working Rules

- Keep notes readable.
- Link generated notes back to source records when available.
- Review AI outputs before using them externally.
- Prefer concise summaries with clear next actions.
